## Blockers

### B1. Remaining orphaned identifiers (~1214 TS2304 errors)

**Files:** `src/transforms/identifiers.ts`, `src/parser/hooks.ts`, `src/emitter/class.ts`

Skip/unknown hook variables are now preserved (fixed 2026-04-11). The remaining TS2304 errors come from:
- **`utility` action hooks** (e.g. `useUniqueId`) — no vars preserved, no import emitted (see S8)
- **Body preamble variables** — non-hook assignments suppressed by emitter (see S14)
- **Imported constants** (e.g. `KeyCode`) — not tracked by the identifier rewriter

**Next steps:** Fix S8 (utility hooks), S14 (body preamble), then audit remaining sources.

---

### B2. Slot/event props excluded from `this.` rewriting (~50 TS2663 errors)

**Files:** `src/transforms/identifiers.ts:40-44`

The member map explicitly skips slot and event props (`if (p.category === 'slot' || p.category === 'event') continue;`). This means when `children`, `className`, `description`, `header`, etc. appear in handler/helper bodies, they aren't rewritten to `this.children`, `this.description`, etc.

**Fix:** Slot props used in non-template contexts (handler bodies, helpers) still need `this.` rewriting. The exclusion should be narrowed to only skip event props in the member map, or slot props should get their own mapping.

---

### B3. `ref.current` pattern persists for non-DOM refs (~46 TS2339 errors)

**Files:** `src/transforms/identifiers.ts:203-209`, `src/emitter/properties.ts:164-180`

When `useRef` is used as a mutable value holder (not DOM ref), the emitter creates `private _fooRef = { current: value }`. But `ref.current` -> `this._fooRef` rewriting eliminates the `.current` access, so code like `fooRef.current = 5` becomes `this._fooRef = 5` (a plain value) while the field is typed as `{ current: ... }`.

**Fix:** Non-DOM refs should either be emitted as plain private fields (not `{ current: ... }` wrappers), or the rewriting should preserve `.current` access.

---

### B4. Shorthand property references to skipped identifiers (~39 TS18004 errors)

**Files:** `src/transforms/identifiers.ts:392`, `src/transforms/cleanup.ts`

Shorthand properties like `{ funnelInteractionId }` reference variables from skipped hooks. Even though those vars are now preserved as class fields (`this._funnelInteractionId`), `isDeclarationPosition` returns `true` for shorthand properties, so the rewriter skips them. The bare name doesn't exist in scope.

**Fix:** Detect shorthand properties whose name matches a member and expand them: `{ funnelId }` -> `{ funnelId: this._funnelId }`.

---

### B5. Modal retains `useEffect` (sole Gate 3 failure)

**Files:** `src/emitter/class.ts:117-122`, `src/transforms/cleanup.ts`

The `modal` component has a private render helper (`_renderPortaledModal`) whose source contains a raw `useEffect(...)` call. Render helpers are extracted as raw source text after JSX transform but *before* hook extraction.

**Fix:** Apply a text-level cleanup pass to render helpers that strips `useEffect(...)` blocks, or process hooks inside render helpers.

---

### B6. State setter functional updates mishandled (~26 TS2322 errors)

**Files:** `src/transforms/identifiers.ts:186-199`

`setFoo(val)` -> `this._foo = val` doesn't handle functional updates: `setFoo(prev => !prev)` becomes `this._foo = prev => !prev` (assigns a function instead of the return value).

**Fix:** Detect functional updater pattern `setFoo(prev => expr)` and rewrite to `this._foo = ((prev) => expr)(this._foo)` or inline the computation.

---

### B7. `fireCancelableEvent` silently downgraded to non-cancelable

**Files:** `src/transforms/events.ts:130-138`

`fireCancelableEvent` is replaced with `fireNonCancelableEvent` at line 137, which silently drops the cancelable semantics.

**Fix:** Preserve `fireCancelableEvent` as its own function call. The regex replacement should use `fireCancelableEvent(this, '${eventName}'` instead of `fireNonCancelableEvent`.

## Small Improvements

### S1. SVG detection TODO in imports

**File:** `src/emitter/imports.ts:143`
**Effort:** Small
**Impact:** Components using inline SVG won't get the `svg` import from `lit`.
**Fix:** Add a `hasSvg()` template walker similar to `hasClassMap()` that checks for `<svg` in template nodes, and emit `collector.addLit('svg')` when found.

---

### S2. `className` not rewritten in JSX path

**File:** `src/transforms/jsx-to-lit.ts:239`
**Effort:** Small
**Impact:** Badge (simplest component) has type errors because `className` isn't `this.className`.
**Fix:** The JSX-to-lit transformer should ensure `className` references in template expressions get the `this.` prefix.

---

### S3. Spread attributes silently dropped

**File:** `src/transforms/jsx-to-lit.ts:198-201`
**Effort:** Medium
**Impact:** Attributes passed via `{...props}` are completely dropped with no warning. Some spreads contain critical ARIA or event attributes.
**Fix:** Emit a `/* TODO: spread ${expr} */` comment in the template, and/or log a warning during transform.

---

### S6. Lifecycle body indentation stripped

**File:** `src/emitter/lifecycle.ts:128-141`
**Effort:** Small
**Impact:** Emitted lifecycle methods lose meaningful indentation from the original source. `line.trimStart()` strips all leading whitespace.
**Fix:** Calculate the base indentation of the source block and strip only that amount, preserving relative indentation.

---

### S8. `useUniqueId` utility action is a no-op

**File:** `src/parser/hooks.ts:148-151`
**Effort:** Small
**Impact:** The `utility` case just returns — no IR produced, no vars preserved, no import emitted. Variables from utility hooks are orphaned.
**Fix:** Implement the utility action: emit a variable initialized with the mapped function call and add the import.

---

### S9. Template literal handling in `splitTopLevel`

**File:** `src/transforms/clsx.ts:440-459`
**Effort:** Small
**Impact:** Template literal expressions with commas inside `${}` would be incorrectly split.
**Fix:** Add backtick tracking to the `splitTopLevel` function, similar to `findMatchingParen`.

---

### S10. Effect cleanup finds first return, not last

**File:** `src/parser/hooks.ts:229-238`
**Effort:** Small
**Impact:** Guard clauses (early returns) may be incorrectly identified as cleanup functions.
**Fix:** Use `findLast` or reverse the search to find the last return statement.

---

### S14. Body preamble completely suppressed

**File:** `src/emitter/class.ts:127-129`
**Effort:** Medium
**Impact:** Computed values, variable assignments, and configuration objects that aren't hooks are silently dropped, contributing to many TS2304 errors.
**Fix:** Selectively emit safe preamble statements (non-hook, non-infrastructure assignments) as class fields or computed getters.

---

### S15. `findMatchingParen` doesn't handle template literals

**File:** `src/transforms/identifiers.ts:452-473`
**Effort:** Small
**Impact:** Setter rewriting could break on backtick arguments containing `)`.
**Fix:** Add template literal tracking as in `clsx.ts:376-393`.

---

### S16. IR `imports` from transforms never used

**File:** `src/emitter/imports.ts:119`
**Effort:** Small
**Impact:** Transform-added imports are silently lost.
**Fix:** In `collectImports()`, iterate `ir.imports` and add each to the collector.
