## Blockers

### B1. Orphaned identifiers (1241x TS2304, 84 components)

"Cannot find name 'xxx'" — variables referenced in handlers, helpers, and effects
that have no declaration in scope. Sources:
- Body preamble variables filtered out by the React-pattern/safety filter
- Imported constants (e.g. `KeyCode`, `warnOnce`) not tracked
- Variables from `inline` action hooks

---

### B2. Slot props missing `this.` prefix (41x TS2663, 33 components)

"Did you mean the instance member 'this.xxx'?" — slot props like `children`,
`header`, `description` aren't in the member map, so they don't get `this.`
prefix in handler/helper bodies.

**Complication:** Simply adding slots to the member map causes new TS2339
errors because slots aren't declared as class fields. Fix requires declaring
slot props as class properties first, then adding to member map.

**Files:** `src/transforms/identifiers.ts:41-43`, `src/emitter/properties.ts`

---

### B3. `ref.current` pattern mismatch (48x TS2339 + 21x TS2345 + 32x TS2322 + 2x TS2741 + 2x TS2353 = 105 errors)

Non-DOM refs emitted as `private _fooRef = { current: value }` but `ref.current`
→ `this._fooRef` rewriting strips `.current`, producing type mismatches:
- TS2339: Property doesn't exist on `{ current: any }`
- TS2345: `{ current: number }` not assignable to `number`
- TS2322: `number` not assignable to `{ current: number }`
- TS2741: Property `current` missing
- TS2353: `cancel` doesn't exist on `{ current: ... }`

**Files:** `src/transforms/identifiers.ts:203-209`, `src/emitter/properties.ts:164-180`
