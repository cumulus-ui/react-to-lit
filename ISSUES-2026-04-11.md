## Remaining Architectural Issues

### HIGH â€” Need deeper changes

#### A1. Parse error swallowed in identifier rewrite (~50-100 errors)

**File:** `src/transforms/identifiers.ts:271-275`

`catch {}` returns original text when ts-morph fails to parse. All identifier
rewriting silently skipped for that block.

**Fix:** Log warning with parse error and offending text.

---

#### A2. REACT_PATTERNS preamble filter patches upstream bugs (~30-60 errors)

**File:** `src/emitter/class.ts:139-145`

The filter removes `useEffect`, `className=`, orphaned `=` from preamble.
Root causes: clsx transform missing className assignments, broken classMap
rewrites producing orphaned assignments.

**Fix:** Fix root causes in clsx transform.

---

#### A3. Spread attributes silently removed (~20-40 errors)

**File:** `src/transforms/cleanup.ts:131`

All spreads dropped with no warning. Includes `{...ariaProps}` with
accessibility attributes.

**Fix:** Warn when spread is removed, or expand known spread objects.

---

#### A4. React catch-all types become `unknown` (~20-50 errors)

**File:** `src/transforms/cleanup-react-types.ts:91-93`

`React.ComponentType`, `React.FC`, etc. become `unknown` with no warning.

**Fix:** Log warning for each React type hitting the catch-all.

---

### MEDIUM â€” Need architectural support

#### Slot/event props not in member map (~130 errors)

Slot props (`header`, `errorText`, `description`, `actions`) and event props
(`onChange`, `onDismiss`) are excluded from the member map. They don't get
`this.` prefix in handler/helper bodies. Fix requires declaring them as class
fields first, which requires emitter changes.

---

#### Relative imports not resolved (~100 errors)

Imported constants/utilities (`KeyCode`, `joinStrings`, `nodeBelongs`) come
from relative Cloudscape modules that don't resolve in the gate2 output.
Import tracking is in place but relative paths are skipped. Fix requires
either monorepo-aware path resolution or declaration stubs.

---

#### Generic type parameter `T` (~35 errors)

Components using `<T>` generic props emit `T` in handler bodies without
declaring the type parameter on the class.

---

#### Ref type mismatches (~103 errors)

Non-DOM refs emitted as `{ current: value }` wrappers but `.current` is
stripped by rewriting. TS2339 + TS2322 + TS2345 + TS2741 + TS2353.

---

### MEDIUM â€” Prop classification

#### A6. `inferFromTypeString` misses object types (~15-30 errors)

**File:** `src/parser/props.ts:298-311`

Object types default to `litType: 'String'`. Should be `attribute: false`.

---

#### A8. Missing type annotation when no default (~10-20 errors)

**File:** `src/emitter/properties.ts:149-158`

Boolean/String/Number props without defaults emit no type annotation.

---

#### A10. Unknown clsx patterns silently dropped (~5-15 errors)

**File:** `src/transforms/clsx.ts:172, 203, 234-235`

Dynamic class expressions silently dropped from classMap.
