## Remaining: 4 components with syntax errors

All original blockers (B1–B7) and small improvements (S1–S16) are resolved.
The last 11 errors are syntax issues in emitted body preamble (TS1005/TS1109/TS1128):

### E1. Untransformed React generic types (9 errors, 2 components)

**Components:** `file-input` (7 errors), `item-card` (2 errors)

Body preamble contains `processAttributes<HTMLDivElement, unknown<HTMLDivElement>>(...)` and similar patterns where a React-specific utility function call with generic type arguments survived into the output. `unknown<T>` is invalid TypeScript.

**Fix:** Filter or rewrite preamble statements containing `unknown<` type patterns, or strip type arguments from `processAttributes` calls.

---

### E2. Cloudscape infra function not filtered (1 error)

**Component:** `date-picker`

`checkControlled('DatePicker', 'value', ...)` is a Cloudscape dev-mode assertion that should not appear in the Lit output.

**Fix:** Add `checkControlled` to the preamble React-pattern filter in `src/emitter/class.ts`.

---

### E3. Orphaned classMap assignment (1 error)

**Component:** `icon`

The clsx transform rewrote `const className = clsx(...)` to `classMap({...})` but lost the variable name, producing a bare `= classMap({...})` statement in the preamble.

**Fix:** The clsx transform should preserve the variable name when rewriting assignments, or the preamble filter should catch orphaned `= ` expressions.
