## Architectural Issues (from codebase audit)

### HIGH — Silent drops that cause errors

#### A1. Parse error swallowed in identifier rewrite (~50-100 errors)

**File:** `src/transforms/identifiers.ts:271-275`

`catch {}` returns original text when ts-morph fails to parse. All identifier
rewriting is silently skipped for that block — `count` stays as `count` instead
of `this._count`.

**Fix:** Log warning with parse error and offending text.

---

#### A2. REACT_PATTERNS preamble filter patches upstream bugs (~30-60 errors)

**File:** `src/emitter/class.ts:139-145`

The filter removes `useEffect`, `className=`, `checkControlled`, orphaned `=`
from preamble. These are symptoms of:
- Hooks inside non-standard statements not extracted by parser (`hooks.ts:70`)
- `className` assignments not caught by clsx transform
- `checkControlled` not removed by cleanup transform
- Orphaned assignments from broken clsx → classMap rewrite

**Fix:** Fix root causes upstream instead of filtering in emitter.

---

#### A3. Spread attributes silently removed (~20-40 errors)

**File:** `src/transforms/cleanup.ts:131`

All spreads dropped with no warning. Includes `{...ariaProps}` with
accessibility attributes.

**Fix:** Warn when spread is removed, or try to expand known spread objects.

---

#### A4. React catch-all types become `unknown` (~20-50 errors)

**File:** `src/transforms/cleanup-react-types.ts:91-93`

`React.ComponentType`, `React.FC`, `React.PropsWithChildren` etc. all become
`unknown` with no warning.

**Fix:** Log warning for each React type hitting the catch-all.

---

### MEDIUM — Incorrect logic

#### A5. Short member names (≤2 chars) skipped in rewrite (~5-10 errors)

**File:** `src/transforms/identifiers.ts:246`

`name.length > 2` check means props like `id` are never rewritten to `this.id`.

**Fix:** Remove or lower the threshold.

---

#### A6. `inferFromTypeString` misses object types (~15-30 errors)

**File:** `src/parser/props.ts:298-311`

Object types, function types, complex unions return `null` → default to
`litType: 'String'`. Should be `attribute: false` for objects.

**Fix:** Add object type detection (`{`, interface names, type literals).

---

#### A7. Props typed `unknown` default to String (~10-30 errors)

**File:** `src/parser/props.ts:125`

Internal-only props without type map entries get `'unknown'` type text →
classified as String.

**Fix:** Log when type is unknown for non-trivial props.

---

#### A8. Missing type annotation when no default (~10-20 errors)

**File:** `src/emitter/properties.ts:149-158`

Boolean/String/Number props without defaults emit no type annotation.

**Fix:** Return type annotation when there's no default value.

---

#### A9. `useContext`/`useReducer` not in standard hooks switch (~10-20 errors)

**File:** `src/parser/hooks.ts:106-128`

These standard React hooks fall through to custom hook lookup → treated as
unknown → stub `any` fields instead of proper context consumption.

**Fix:** Add `useContext` and `useReducer` cases to the switch.

---

#### A10. Unknown clsx patterns silently dropped (~5-15 errors)

**File:** `src/transforms/clsx.ts:172, 203, 234-235`

Dynamic class expressions, function call results silently dropped from classMap.

**Fix:** Emit `/* TODO: unhandled clsx arg */` comment instead of dropping.

---

### LOW — Defensive fixes

#### A11. Missing `preservedVars` in fallback hook result (crash risk)

**File:** `src/parser/index.ts:118`

Fallback result missing `preservedVars: []` — causes TypeError for expression-body
components.

**Fix:** Add `preservedVars: []` to fallback object.

---

#### A12. Empty catch on index component parsing (~5-10 errors)

**File:** `src/parser/index.ts:131`

All parsing failures swallowed. Public methods from `useImperativeHandle` in
index.tsx silently lost.

**Fix:** Catch only "no component found", warn for unexpected errors.

---

#### A13. Helper parse error swallowed (~5-10 errors)

**File:** `src/emitter/class.ts:205-206`

If helper can't be parsed, emitted raw with wrong structure.

**Fix:** Log warning.

---

#### A14. Hooks in non-standard statements missed (~5-15 errors)

**File:** `src/parser/hooks.ts:70-86`

Hooks inside `if` blocks, `try/catch`, or `for` loops silently skipped.

**Fix:** Track unprocessed statements containing `use` calls.

---

#### A15. Controller/context hook silently dropped (~2-5 errors)

**File:** `src/parser/hooks.ts:139-147`

If mapping exists but `mapping.controller` is undefined or no `decl`, hook
is invisible — not in skipped, unknown, or controllers.

**Fix:** Add to `result.skipped` when mapping can't be processed.

---

#### A16. Default branch missing on hook action switch

**File:** `src/parser/hooks.ts:133-154`

No `default` case. New action types would silently do nothing.

**Fix:** Add `default: result.unknown.push(hookName);`

---

#### A17. Effect dedup ignores deps (~1-3 errors)

**File:** `src/emitter/lifecycle.ts:21-26`

Two effects with same body but different deps have one silently removed.

**Fix:** Include `deps` in the dedup key.

---

#### A18. useState with <2 elements dropped (~1-3 errors)

**File:** `src/parser/hooks.ts:176`

`const [value] = useState(0)` (read-only state) silently lost.

**Fix:** Process even with 1 element.
